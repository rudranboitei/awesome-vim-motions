# 🚀 The Ultimate VS Code Vim Motions Guide

Learning Vim motions is one of the highest-leverage skills you can build as a software engineer. Instead of pointing and clicking your mouse, Vim lets you edit code at the speed of your thoughts. 

This guide teaches you Vim step-by-step using **real-world developer scenarios** instead of abstract keys.

---

<InfoBox type="tip">
  **The Golden Rule:** You don't need to learn 100+ commands. Master just 15-20 commands to get 80% of the productivity gains.
</InfoBox>

## ⚙️ Custom VS Code Vim Configuration

Rather than running raw Vim in a terminal, we can use the **VSCodeVim** extension. This allows us to keep our VS Code plugins, themes, and debuggers, while gaining the speed of Vim.

To get started, add these premium mappings to your `settings.json`:

```json
{
  "vim.useSystemClipboard": true,
  "vim.leader": " ",
  
  // Escape home row
  "vim.insertModeKeyBindings": [
    { "before": ["j", "k"], "after": ["<Esc>"] },
    { "before": ["k", "j"], "after": ["<Esc>"] }
  ],
  
  // Custom navigation
  "vim.normalModeKeyBindingsNonRecursive": [
    // Leader actions
    { "before": ["<leader>", "w"], "commands": ["workbench.action.files.save"] },
    { "before": ["<leader>", "q"], "commands": ["workbench.action.closeActiveEditor"] },
    { "before": ["<leader>", "f"], "commands": ["workbench.action.quickOpen"] },
    { "before": ["<leader>", "e"], "commands": ["workbench.view.explorer"] },
    { "before": ["<leader>", "n"], "commands": ["explorer.newFile"] },
    
    // Tab switching (Buffer style)
    { "before": ["H"], "commands": ["workbench.action.previousEditor"] },
    { "before": ["L"], "commands": ["workbench.action.nextEditor"] },
    
    // Split pane jumps
    { "before": ["<C-h>"], "commands": ["workbench.action.focusLeftGroup"] },
    { "before": ["<C-l>"], "commands": ["workbench.action.focusRightGroup"] },
    { "before": ["<C-j>"], "commands": ["workbench.action.focusBelowGroup"] },
    { "before": ["<C-k>"], "commands": ["workbench.action.focusAboveGroup"] }
  ]
}
```

## 📚 Chapter 1: Basic hjkl Movements

Vim uses different **modes** to separate editing from navigation:
*   **NORMAL mode** (`Esc` or custom `jk`): Used for jumping around, searching, and refactoring.
*   **INSERT mode** (`i`): Used for typing text like a normal editor.
*   **VISUAL mode** (`v`): Used for selecting blocks of text.

Instead of arrow keys, navigate in Normal Mode using your home row:

```
      ▲ k (Up)
◄ h (Left)   l (Right) ►
      ▼ j (Down)
```

### 📝 Scenario: You have a typo on the line above
*   **Without Vim:** Grab mouse ➔ click typo ➔ fix ➔ click back.
*   **With Vim:** `jk` (exit insert) ➔ `k` (up) ➔ `h/l` (move to typo) ➔ `r` (replace character).

## 📚 Chapter 2: Word & Line Jumping

Moving character-by-character is slow. Let's start jumping.

### Word Jumping
*   `w` ➔ Jump to the **start** of the next word.
*   `b` ➔ Jump **back** to the start of the previous word.
*   `e` ➔ Jump to the **end** of the current or next word.

### Line Jumping
*   `0` ➔ Jump to the absolute **start** of the line.
*   `^` ➔ Jump to the **first text character** (skipping indentation).
*   `$` ➔ Jump to the **end** of the line.

<CodeGroup title="Practice Navigation">
```javascript
// Cursor starts on const
const totalPrice = calculateSum(item1, item2, item3);
// Use 'w' to jump word-by-word
// Use '$' to jump straight to the semicolon
```

## 📚 Chapter 3: File Navigation & Search

Finding text quickly across a large file is key to keeping your hands on the keyboard.

### Jump in File
*   `gg` ➔ Jump to the **top** of the file.
*   `G` ➔ Jump to the **bottom** of the file.
*   `Ctrl + d` ➔ Scroll **down** half a page.
*   `Ctrl + u` ➔ Scroll **up** half a page.
*   `:LineNumber` ➔ Jump to a specific line (e.g. `:42` jumps to line 42).

### Search
*   `/searchTerm` ➔ Search forward in the file. Hit `Enter`, then press `n` for next match, and `N` for previous match.
*   `*` ➔ Search forward for the word currently under the cursor.

## 📚 Chapter 4: Core Editing & Text Objects

This is where Vim becomes a superpower. By combining actions with **Text Objects**, you can edit complex syntax structures in seconds.

### The Operators
*   `d` ➔ Delete (Cut)
*   `c` ➔ Change (Delete and enter INSERT mode)
*   `y` ➔ Yank (Copy)
*   `v` ➔ Select (Visual)

### The Inner Delimiters
Combine `c` or `d` with `i` (inner) or `a` (around) and a delimiter:

| Command | Action | React / Web Dev Use Case |
| :--- | :--- | :--- |
| `ciw` | Change Inner Word | Rename a variable at the cursor |
| `ci"` | Change inside double quotes | Rewrite string prop: `className="[edit here]"` |
| `ci(` | Change inside parentheses | Re-write function parameters or hooks arguments |
| `ci{` | Change inside curly braces | Edit object properties or JSX expressions |
| `cit` | Change inside JSX/HTML tag | Change tag text content: `<h1>[change text]</h1>` |

### 📝 Scenario: Rename React hook argument
```javascript
const [user, setUser] = useState(initialUserData);
//                                     ▲
// Cursor anywhere on initialUserData ➔ Type 'ci(' ➔ Type new value
```

## 📚 Chapter 5: Visual Mode & Block Selections

Sometimes you need a visual representation of what you are editing.
*   `v` ➔ Character selection.
*   `V` ➔ Line selection.
*   `Ctrl + v` ➔ Visual Block (column) selection.

### 📝 Scenario: Commenting out multiple lines
1. Move to the first character of line 1.
2. Press `Ctrl + v` to enter Visual Block mode.
3. Move down (`j j j`) to select the first column of the lines.
4. Press `Shift + i` (Insert at start).
5. Type `// ` and then press `Esc`. The comment will apply to all lines.

## 📚 Chapter 6: The Magic Dot Operator (`.`)

The dot operator (`.`) repeats your last edit action. This is the foundation of fast file refactoring.

### 📝 Scenario: Renaming `usreName` to `userName` across a file
1. Search for the typo: `/usreName` + `Enter`.
2. Change the first instance: `ciw` ➔ `userName` ➔ `Esc`.
3. Jump to the next occurrence: `n`.
4. Apply the exact same change: `.`.
5. Repeat `n` and `.` until all typos are fixed.

## 🎓 The 4-Week Learning Roadmap

To prevent overwhelm, follow this step-by-step integration:

### 📅 Week 1: Mouse-Free Movement
*   Disable your arrow keys in settings.
*   Practice only using `h j k l` and `w b e` to navigate files.
*   *Goal:* Never touch the mouse to position your cursor.

### 📅 Week 2: Editing Combos
*   Focus on replacing manual backspacing with change and delete.
*   Force yourself to use `ciw`, `ci"`, `cit`, and `dd`.
*   *Goal:* Never double-click to select a word.

### 📅 Week 3: Search & Repeat
*   Use `/` to move to words instead of scroll-hunting.
*   Incorporate the dot operator (`.`) to speed up duplicate edits.
*   *Goal:* Refactor multiple instances of code under 10 seconds.

### 📅 Week 4: Multi-split Layouts
*   Configure the split editor bindings (`Ctrl+h/j/k/l`).
*   Utilize tab cycling (`H` and `L`) to navigate multiple active files.
*   *Goal:* Code, save, close, and navigate tabs with zero mouse interaction.
